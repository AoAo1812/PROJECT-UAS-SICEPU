# Plan: OTP Verification + Backup Email untuk Keamanan Akun

## Ringkasan
Menambahkan verifikasi OTP via email sebelum ubah password, dan sistem email backup wajib saat registrasi untuk account recovery.

## Flow Keamanan Baru

### Flow Ubah Password (dengan OTP)
1. User isi form ubah password (current + new + confirm)
2. Klik "Ubah Password" → backend kirim OTP 6 digit ke email utama
3. Muncul modal/input OTP (6 digit)
4. User masukkan OTP → backend verifikasi
5. Jika OTP valid → password diubah
6. OTP expired setelah 5 menit, max 3 attempts

### Flow Registrasi (dengan Backup Email)
1. User isi: nama, email, password, confirm password
2. **Input baru: Email Backup (wajib)** — email kedua yang beda dari email utama
3. Email backup disimpan di database user
4. Nanti bisa dipakai untuk account recovery

### Flow Account Recovery (Optional/Future)
- Jika akun utama terkompromi, user bisa minta reset via email backup
- OTP dikirim ke email backup

### 4. `lib/email.ts` — **BARU**
Utility Nodemailer:
- `createTransporter()` — buat transporter dengan Gmail SMTP
- `sendOTP(to: string, code: string)` — kirim email berisi kode OTP 6 digit
- Template email HTML yang rapi dengan kode OTP besar

### 5. `lib/otp-store.ts` — **BARU**
In-memory OTP storage (karena project pakai JSON files, ini paling simpel):
- `storeOTP(email, code)` — simpan OTP + expiry (5 menit) + attempts (max 3)
- `verifyOTP(email, code)` — verifikasi + hapus jika valid atau expired
- `cleanupExpired()` — auto-clean OTP expired

### 6. `app/api/auth/otp/send/route.ts` — **BARU**
Endpoint kirim OTP:
- POST { email } → generate 6-digit code → simpan ke store → kirim via email
- Return success message (jangan return code di response untuk security)
- Rate limit: jangan kirim OTP berulang dalam 60 detik

### 7. `app/api/auth/otp/verify/route.ts` — **BARU**
Endpoint verifikasi OTP:
- POST { email, code } → cek store → return valid/invalid
- Handle expiry + attempt limit
