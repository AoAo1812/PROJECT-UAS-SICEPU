# SICEPU Fixes Plan: Google Login + Chat Bot + Logo

## Issue 1: Google Login Tidak Berfungsi

### Root Cause
Project punya **2 sistem auth yang saling terpisah**:
- **Sistem A (aktif)**: Custom JWT via cookie `token` тАФ dipakai middleware, dashboard layout, `/api/auth/me`
- **Sistem B (NextAuth v5 beta)**: Google OAuth тАФ menghasilkan cookie `next-auth.session-token`

Setelah Google login, NextAuth set cookie `next-auth.session-token`, tapi middleware hanya cek cookie `token` тЖТ **infinite redirect loop** ke `/login`.

### Bugs Tambahan
- Tidak ada file `.env` (hanya `.env.example` dengan nama var salah untuk v5)
- `AUTH_SECRET` tidak ada тЖТ NextAuth v5 tidak bisa jalan
- `next-auth` masih versi beta.31 (tidak stabil)
- `google-callback/route.ts` adalah dead code
- Hardcoded JWT secret: `sicepu-secret-key-2026`

### Strategy: Bridge NextAuth тЖТ Custom JWT
Setelah Google login berhasil, generate custom JWT dan set cookie `token` (yang dipakai seluruh app).

### Changes

#### 1.1 `lib/auth-config.ts` тАФ tambah `redirect` callback
Redirect ke bridge route setelah Google auth berhasil.

#### 1.2 **NEW** `app/api/auth/google-complete/route.ts` тАФ bridge route
- Panggil `auth()` dari NextAuth untuk dapat session
- Generate custom JWT (dari `lib/auth.ts`)
- Set cookie `token` via `Set-Cookie` header
- Redirect ke `/dashboard` atau `/admin` berdasarkan role

#### 1.3 Delete `app/api/auth/google-callback/route.ts` тАФ dead code

#### 1.4 **NEW** `.env` file
```
AUTH_URL=http://localhost:3000
AUTH_SECRET=<auto-generate>
GOOGLE_CLIENT_ID=<user must fill after setup>
GOOGLE_CLIENT_SECRET=<user must fill after setup>
JWT_SECRET=<auto-generate>
```

#### 1.5 Update `.env.example` тАФ fix var names
`NEXTAUTH_URL` тЖТ `AUTH_URL`, `NEXTAUTH_SECRET` тЖТ `AUTH_SECRET`

#### 1.6 `next.config.ts` тАФ tambah Google avatar ke remotePatterns

#### 1.7 **Panduan Google OAuth Setup** (diberikan ke user):
1. Buka console.cloud.google.com
2. Buat project baru (atau pilih project yang ada)
3. Buka API & Services тЖТ Library тЖТ cari "Google+ API" atau "People API" тЖТ Enable
4. Buka API & Services тЖТ Credentials тЖТ Create Credentials тЖТ OAuth client ID
5. Application type: **Web application**
6. Authorized redirect URIs: tambah `http://localhost:3000/api/auth/callback/google`
7. Copy **Client ID** dan **Client Secret** тЖТ paste ke `.env`

## Issue 2: Chat Bot Auto-Notification

### Current State
Auto-notification **sudah ada** di backend:
- `app/api/reports/[id]/route.ts:71-84` тАФ Status changes trigger bot messages
- `app/api/reports/route.ts:109-118` тАФ Welcome message on creation
- `app/api/chats/route.ts:7-29` тАФ Bot reply templates per status

### Problem
**ChatPanel hanya fetch messages sekali saat mount** (line 32-37). Tidak ada polling. User harus refresh page untuk melihat pesan bot baru.

### Fix: Add polling ke `components/chat/ChatPanel.tsx`
Tambah `useEffect` dengan `setInterval` re-fetch messages setiap 5 detik тЖТ user langsung lihat pesan bot tanpa refresh.
